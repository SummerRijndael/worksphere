System Overview

The Coresync V2 FAQ / Knowledge Base system provides a self-service help center for users. It supports Categories and Articles with rich text content, voting (helpful/unhelpful), comments, and an admin management interface with sorting, filtering, and pagination.

API Endpoint Map

Method Endpoint Description
Admin Endpoints
GET /api/admin/faq/categories List categories (paginated, sortable, date range)
POST /api/admin/faq/categories Create category
PUT /api/admin/faq/categories/{id} Update category
DELETE /api/admin/faq/categories/{id} Delete category (soft delete)
GET /api/admin/faq/articles List articles (paginated, sortable, date range)
POST /api/admin/faq/articles Create article
PUT /api/admin/faq/articles/{id} Update article
DELETE /api/admin/faq/articles/{id} Delete article (soft delete)
GET /api/admin/faq/stats Get FAQ stats (views, votes, most viewed)
Public Endpoints
GET /api/faq/categories List public categories
GET /api/faq/articles/{slug} Get article by slug
POST /api/faq/articles/{slug}/vote Vote (helpful/unhelpful)

Core Features

  • Rich Text Editor TipTap-powered article editing with formatting, images, and code blocks.
  • Dynamic Sorting Sort by any column (name, views, votes, date, author, category).
  • Date Range Filtering Filter categories and articles by creation date range.
  • Voting System Helpful/Unhelpful voting with visual score indicators.
  • State Persistence User preferences (tab, view mode, sort order) persist across sessions.
  • List & Grid Views Toggle between table and card layouts for both tabs.

Limitations & Future Todo

  • Full-text Search: Basic SQL search implemented (Review Scout for scaling).
  • Article Versioning: Revision history and restore functionality (Planned).
  • Hard Delete: Currently only Soft Delete is supported. Hard delete requires implementing manual media cleanup to avoid orphaned files.
  • SEO Metadata: Custom meta descriptions per article not yet supported.
