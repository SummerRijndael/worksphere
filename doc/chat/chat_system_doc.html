System Overview

The Coresync V2 Chat System is a real-time messaging engine built on Laravel Reverb (WebSockets) and Vue 3. It supports 1-on-1 Direct Messages (DMs) and multi-user Group Chats with file attachments, read receipts, and typing indicators.

Backend Architecture

Controller Layer

app/Http/Controllers/Api/ChatApiController.php

app/Http/Controllers/Api/Chat/GiphyController.php

Handling 22+ API endpoints. Validates requests, authorizes actions via Policies, and returns standardized JSON responses.

Service Layer (The Engine)

app/Services/ChatEngine.php

app/Services/Chat/GiphyService.php

Core logic domain. Handles message persistence, sanitization (HTML entities), read status calculation, and response normalization.

Event System

app/Services/ChatEvents.php & app/Events/*

Abstracts broadcasting. Events: `MessageCreated`, `MessageRead`, `UserTyping`, `ChatBadgeUpdated`.

Asynchronous Jobs

app/Jobs/ProcessChatMessage.php

Offloads heavy lifting (broadcasting, unread count updates) to the queue to keep the API fast.

Database Schema

chats

  • id (int)
  • public_id (ulid)
  • type (enum: dm/group)
  • name (string, nullable)

chat_messages

  • id (int)
  • chat_id (fk)
  • user_id (fk)
  • content (text)
  • metadata (json, nullable)
  • reply_to_message_id (fk)

chat_participants

  • chat_id (fk)
  • user_id (fk)
  • role (enum: owner/member)
  • last_read_message_id (int)

Core Features

  • Real-time Messaging Users receive messages instantly via WebSocket without reloading.
  • Optimistic UI Messages appear instantly in the UI before server confirmation.
  • Read Receipts & Unread Counts Tracks per-user read status and badging.
  • Media & File Sharing Upload images/docs with image optimization.

Limitations & Future Todo

  • Push Notifications: Not yet integrated (mobile/web push).
  • End-to-End Encryption: Messages are stored in plaintext in the database (standard HTTPS encryption in transit).
  • Reactions: Emoji reactions to messages are planned but not implemented.
  • Message Editing: Currently messages are immutable after sending.
